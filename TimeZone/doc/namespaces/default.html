Classes

ConfigManager Reads from the local XML configuration files and sends it to the utility controller
DBController Controls the setting up of the database, calls operations from the classes in the DB component and provides the ulility controller with data
Logger Handles logging of events in the application
medoo
MyDB Inherits the Medoo framework. Acts as an abstracted mediating layer in the application.
MySQLDB Inherits from class MyDB. Stores variables and handle operations that are specific for the given MySQL database
SessionManager Handles the session for the runtime of the script. Does this by dropping a file at a given location. If the file exists, the script is running.
StampFetch Handles the fetching, checking, converting and returning of all the individual timestamps
TimeComm Handles the communication with the web service. Checks if the domain at the given URL is available and that there exists a JSON there
TimeController Validates availability of domain and JSON, arranges fetching of timezones and timestamps and sends records of errors occurred to the utility controller
TimeView Handles the updating of the view with the dynamic content
UtilController Sets up the configuration for the script and takes the the outup from the time component side and provides it as input for the the database component side.
ZoneFetch Fetches the timezones from the web service
